# Moment Generator

Moment Generator is a Qt + KaTeX desktop tool for deriving exact finite-sample bias formulas of sample central moments via integer partitions.

Given the moment order `k`, the program enumerates every integer partition of `k` without parts equal to `1`, computes the exact coefficient of each population central-moment product, and renders the formula

```tex
\mathbb{E}[M_k]
=
\sum_{\lambda \vdash k,\;1\notin\lambda}
c_\lambda(n)\prod_q v_q^{m_q}.
```

The current version focuses on formula generation and inspection:

- integer partitions without `1`;
- exact symbolic coefficient formulas in `n` for `E[M_k]`;
- exact product-moment systems `E[M_k-vector] = A_k(n) V_k-vector` from cached generator output;
- pre-generated product-system caches for `2 <= k <= 20`;
- on-demand cache generation for `21 <= k <= 25` with a warning before the first calculation;
- optional numeric coefficient evaluation for a chosen sample size;
- KaTeX preview through Qt WebEngine;
- copyable LaTeX output for paper writing.

## Build

This project is set up for the local Qt installation:

```powershell
D:\Qt\6.9.3\msvc2022_64\bin\qt-cmake.bat -S . -B build -G Ninja
D:\Qt\Tools\CMake_64\bin\cmake.exe --build build
```

If MSVC is not already available in the shell, run the commands from a Visual Studio Developer PowerShell or open the folder in Qt Creator with the `Qt 6.9.3 msvc2022_64` kit.

## Notes

The formula renderer currently loads KaTeX from jsDelivr. A later version can vendor KaTeX assets under `resources/katex/` for offline rendering.
## Product-system cache

The full unbiased product-term system is generated by:

```powershell
python tools\generate_product_system_cache.py --min-order 2 --max-order 20 --output-dir cache\product_systems
```

Cache files store exact entries of `A_k(n)` as sums of terms of the form
`coeff * falling(n, b) * n^p`, where `falling(n, b) = n(n-1)...(n-b+1)` and `falling(n, 0)=1`. Arbitrary-size integer coefficients are saved as strings.
The Qt app reads these files instead of recomputing high-order systems in the UI.

Orders `2..20` are shipped as precomputed cache files. Orders `21..25` are supported but are not shipped precomputed; the app warns before first calculation and writes the generated cache to the user app-data cache directory.
